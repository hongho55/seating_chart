# 기준안 적용 기능 계획

## 핵심 개념
- `기준안`은 랜덤의 출발점(seed)이 아니라 **완성된 최종 배치안**이다.
- 평소 일반 화면은 비어 있거나 현재 live 배치를 보여줄 수 있지만, `기준안 저장` 자체가 곧바로 발표 화면을 덮어쓰는 의미는 아니다.
- 수업/발표 시에는 `기준안 적용`을 켜고 **자리 배정 시작**을 누르면, 저장된 기준안을 그대로 공개한다.
- 기준안은 **100% 재현**되어야 하며, 한 번 적용 후에는 `기준안 적용` 상태를 자동 해제할 수 있다.

## 목표
- 반 버튼 옆 `⋯` 메뉴에서 `기준 배치 편집` 진입
- 편집 화면 상단에서만 `전체 랜덤`, `미고정만 랜덤`, `기준 배치 저장`, `취소` 제공
- 반마다 기준안 1개만 유지
- 일반 모드의 routine seating edits는 저장된 기준안을 암묵적으로 바꾸지 않음
- `기준안 적용` 토글 ON 상태에서 `자리 배정 시작`을 누르면 저장된 기준안을 그대로 공개
- 기준안 적용이 끝나면 토글/armed 상태는 자동 해제 가능
- 한 파일에 기능을 몰아넣지 않고 작은 단계로 분리 구현

## UX 확정안
### 일반 모드
- 반 선택 버튼 옆 `⋯`
- 메뉴 항목: `반 정보 수정`, `학생 명단 수정`, `기준 배치 편집`
- 메인 화면에는 작은 토글/상태 표시 제공
  - 예: `기준안 적용`
  - ON이면 작은 상태 표시: `기준안 적용 예정`
- 메인 실행 버튼은 장기적으로 `자리 배정 시작`으로 가는 것이 자연스럽다.
  - 단기적으로는 기존 버튼을 유지하더라도, 기준안 적용 ON일 때 내부 동작은 **완전 재현 모드**로 분기한다.

### 기준 배치 편집 모드
상단 액션만 노출:
- 취소
- 전체 랜덤
- 미고정만 랜덤
- 기준 배치 저장

편집 모드에서 가능:
- 학생 직접 배치
- 두 학생 클릭 후 자리 바꾸기
- 고정(pin)
- 랜덤 반복

### 기준안 적용 동작
- `기준안 적용` ON
- `자리 배정 시작` 클릭
- 저장된 기준안이 그대로 공개됨
- 기존 공개 애니메이션/발표 흐름이 있다면 그 흐름을 재사용
- 적용 완료 후 `기준안 적용`은 자동 OFF 가능

## 데이터 구조 초안
- `classroom.basePlan`: 저장된 기준안 (반별 1개)
- `classroom.basePlanArmed` 같은 영속 필드는 신중히 검토
  - 가능하면 발표 직전 토글은 앱/UI 상태로 두고, 굳이 영구 저장하지 않는 쪽을 우선 고려
- 일반 live 배치와 saved basePlan은 분리 유지
- 학생 삭제/백업 로드/레거시 데이터 정리 시에는 saved basePlan도 안전하게 sanitize

## 현재 구현과 충돌하는 부분
### 이미 맞는 부분
- `basePlan` 데이터 구조 존재
- 기준 배치 편집 진입 UI 존재
- 편집 상단 액션 바 존재
- saved basePlan과 live seating 분리 작업 완료

### 바꿔야 하는 부분
1. **현재 일반 랜덤은 saved basePlan을 출발점으로 사용함**
   - 새 요구사항에서는 이것보다 `기준안 적용 ON일 때 완전 재현`이 더 중요함
   - 즉 `기준안 적용`과 `일반 랜덤`은 다른 모드여야 함

2. **기준안 저장 후 일반 화면에서 바로 ‘그대로 적용’되는 흐름이 아님**
   - 현재는 저장된 기준안이 발표 시 자동 재현되는 UX가 없음
   - `기준안 적용` 토글 + 실행 시 분기 필요

3. **기존 버튼 명/행동 의미가 모호함**
   - `조건 반영 랜덤 배치`는 기준안 완전 재현과 개념이 다름
   - 최소한 ON/OFF에 따라 버튼 동작을 분기하거나, 장기적으로 버튼 문구를 정리해야 함

## 분할 구현 원칙
1. 한 단계당 한 가지 축만 변경
2. App.tsx에는 모드 전환/화면 조합만 두고 상태 로직은 lib로 분리
3. 새 기능은 가능하면 아래처럼 분산
   - `src/lib/basePlanState.ts`
   - `src/lib/classroomActions.ts`
   - `src/lib/revealMode.ts` 또는 유사 helper
   - `src/components/...`
4. 커밋 단위마다 빌드/푸시
5. `기준안 적용` 토글, 적용 실행, 적용 후 해제는 한 번에 거대하게 얽지 말고 단계별로 나눔

## 다음 구현 순서
### 5단계
- `기준안 적용` 토글/상태 추가
- 일반 랜덤과 기준안 완전 재현 모드 분기
- ON일 때는 saved basePlan을 그대로 사용하는 실행 경로 추가
- 적용 후 OFF 처리

### 6단계
- 기준안 적용 모드의 문구/버튼 정리
- 필요하면 `조건 반영 랜덤 배치` → `자리 배정 시작`으로 정리

### 7단계
- 실제 공개 애니메이션/단계 공개 흐름과 기준안 완전 재현 연결
- 기존 애니메이션이 있으면 재사용, 없으면 과도한 신규 시스템 없이 최소 구현 검토

## Codex 작업 원칙
- 한 번에 거대한 기능 추가 금지
- 새 파일로 분리 우선
- 빌드 산출물 커밋 금지
- 기준안 완전 재현과 일반 랜덤을 혼동하지 말 것
