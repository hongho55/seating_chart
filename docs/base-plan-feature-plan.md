# 기준 배치 기능 계획

## 목표
- 반 버튼 옆 `⋯` 메뉴에서 `기준 배치 편집` 진입
- 편집 화면 상단에서만 `전체 랜덤`, `미고정만 랜덤`, `기준 배치 저장`, `취소` 제공
- 반마다 기준 배치 1개만 유지
- 일반 랜덤은 기준 배치를 출발점으로 사용
- 한 파일에 기능을 몰아넣지 않고 작은 단계로 분리 구현

## UX 초안
### 일반 모드
- 반 선택 버튼 옆 `⋯`
- 메뉴 항목: `반 정보 수정`, `학생 명단 수정`, `기준 배치 편집`
- 일반 화면은 지금 흐름 최대한 유지
- 필요하면 작은 상태 표시만 추가: `기준 배치 설정됨`

### 기준 배치 편집 모드
상단 액션만 노출:
- 취소
- 전체 랜덤
- 미고정만 랜덤
- 기준 배치 저장

편집 모드에서 가능:
- 학생 직접 배치
- 두 학생 클릭 후 자리 바꾸기
- 고정(pin)
- 랜덤 반복

## 데이터 구조 초안
- classroom에 `basePlan` 또는 이에 준하는 필드 추가
- basePlan은 반별 1개만 유지
- 포함 대상:
  - seats
  - groups
  - layoutConfig
  - lastViewMode는 제외 가능
- 일반 랜덤 시 현재 classroom이 아니라 basePlan 기반으로 좌석 시작 상태 구성

## 분할 구현 원칙
1. 한 단계당 한 가지 축만 변경
2. App.tsx에는 모드 전환/화면 조합만 두고 상태 로직은 lib로 분리
3. 새 기능은 가능하면 아래처럼 분산
   - `src/lib/basePlanState.ts`
   - `src/lib/classroomActions.ts` 확장 (필요 최소만)
   - `src/components/...` 또는 패널 컴포넌트 분리
4. 커밋 단위마다 빌드/푸시

## 구현 순서
### 1단계
- 기준 배치 데이터 구조 추가
- 저장/로드/백업 파서 호환
- 동작 변화는 최소화

### 2단계
- 반 버튼 옆 `⋯` 메뉴에 `기준 배치 편집` 진입 추가
- 편집 모드 상태만 연결
- 아직 랜덤/저장은 최소 연결

### 3단계
- 편집 모드 상단 액션 추가
- `전체 랜덤`, `미고정만 랜덤`, `기준 배치 저장`, `취소`

### 4단계
- 일반 랜덤이 기준 배치 기반으로 동작하도록 연결
- 수동 테스트

## Codex 작업 원칙
- 한 번에 거대한 기능 추가 금지
- 새 파일로 분리 우선
- 빌드 산출물 커밋 금지
