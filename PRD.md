# 자리 배치 웹앱 PRD

## 1. 제품 개요

과학 전담 교사처럼 여러 학년·여러 반을 관리하는 교사가, 다양한 교실 구조에 맞게 좌석을 만들고 배치하며 저장·출력할 수 있는 웹앱이다.

이 앱은 단순 랜덤 추첨기가 아니라, 교사가 의도적으로 통제할 수 있는 자리 배치 도구를 목표로 한다.

## 2. 해결하려는 문제

- 반마다 좌석 구조가 다르다.
- 1인 시험형, 2인 짝, 4인 모둠, 6인 모둠을 모두 다뤄야 한다.
- 학생 간 금지 조합, 고정 좌석, 성별 기준 같은 조건이 있다.
- 교사에게 보이는 방향과 학생에게 보이는 방향이 다르다.
- 여러 반의 배치를 저장하고 다시 불러와야 한다.
- 최종 배치를 교사용과 학생용으로 각각 인쇄해야 한다.

## 3. 주요 사용자

- 초등·중등 교사
- 전담 교사
- 담임 교사

## 4. 핵심 사용 시나리오

### 시나리오 A. 새 반 자리 만들기

1. 교사가 학년, 반, 교실 이름을 선택한다.
2. 좌석 프리셋을 고른다.
3. 예: `6인 모둠 4개`를 클릭하면 24석이 자동 생성된다.
4. 필요한 경우 좌석을 직접 드래그해서 위치를 조정한다.
5. 여러 좌석을 선택해 하나의 모둠으로 묶고 테두리를 표시한다.

### 시나리오 B. 조건을 넣고 랜덤 배치하기

1. 학생 명단을 불러온다.
2. 일부 학생은 고정 좌석으로 지정한다.
3. 만나면 안 되는 학생 조합을 설정한다.
4. 성별 배치 모드를 선택한다.
5. 랜덤 배치를 실행한다.
6. 규칙 충돌이 있으면 경고를 확인하고 다시 배치한다.

### 시나리오 C. 저장 후 다시 사용하기

1. 반별 좌석 배치를 저장한다.
2. 다음 수업 때 저장된 배치를 다시 불러온다.
3. 이전 배치를 복사해서 일부만 수정한다.

### 시나리오 D. 인쇄하기

1. 교사 관점 또는 학생 관점을 선택한다.
2. 교사용 인쇄 또는 학생용 인쇄를 누른다.
3. 종이 출력 또는 PDF 저장을 한다.

## 5. 기능 요구사항

## 5.1 학생 / 반 관리

- 학년, 반, 교실명 단위로 데이터를 저장할 수 있어야 한다.
- 한 교사가 여러 학년·여러 반의 좌석 배치를 관리할 수 있어야 한다.
- 학생 명단을 직접 입력할 수 있어야 한다.
- 학생별 속성을 저장할 수 있어야 한다.

학생 속성 예시:

- 이름
- 번호
- 성별
- 결석 여부
- 메모
- 고정 필요 여부

## 5.2 좌석 레이아웃 편집

- 좌석을 직접 추가, 이동, 삭제할 수 있어야 한다.
- 마우스 드래그로 좌석과 학생을 이동할 수 있어야 한다.
- 여러 좌석을 선택할 수 있어야 한다.
- 여러 좌석을 선택한 뒤 하나의 모둠으로 묶을 수 있어야 한다.
- 모둠은 테두리, 배경색, 라벨로 시각적으로 구분되어야 한다.

### 좌석 프리셋

- 1인 시험형
- 2인 짝꿍형
- 4인 모둠형
- 6인 모둠형

### 프리셋 생성 예시

- 2인 책상 12개 생성
- 4인 모둠 6개 생성
- 6인 모둠 4개 생성

### 직접 편집 기능

- 좌석 위치 미세 조정
- 행/열 간격 조정
- 모둠 회전
- 교탁 위치 표시
- 칠판 방향 설정

## 5.3 보기 모드

- `교사 관점`과 `학생 관점`을 버튼으로 전환할 수 있어야 한다.
- 관점 전환 시 좌석 배치가 교실 기준으로 반대 방향에서 보이도록 회전되어야 한다.
- 인쇄 시에도 동일하게 `교사용`, `학생용` 두 가지 버전을 지원해야 한다.

## 5.4 자동 배치 / 랜덤 배치

- 완전 랜덤 배치
- 일부 학생 고정 후 나머지 랜덤 배치
- 이전 배치와 너무 비슷하지 않게 재배치
- 교사가 미리 정한 구조를 유지한 채 랜덤처럼 보이게 하는 연출용 배치

### 성별 배치 모드

랜덤 배치 시 성별 규칙을 선택할 수 있어야 한다.

- `랜덤`: 성별과 무관하게 배치
- `동성 우선`: 가능한 한 같은 성별끼리 짝 또는 같은 모둠으로 구성
- `이성 우선`: 가능한 한 다른 성별이 섞이도록 짝 또는 모둠 구성

성별 배치 모드 규칙:

- 2인 좌석에서는 짝 기준으로 적용한다.
- 4인, 6인 모둠에서는 모둠 내부 구성 기준으로 적용한다.
- 학생 성별 정보가 없으면 해당 학생은 `무작위 배치 대상`으로 처리한다.
- 현재 학생 구성상 조건을 완전히 만족할 수 없으면, 앱은 가능한 최선의 결과를 배치하고 부족한 조건 수를 경고해야 한다.

### 랜덤 배치 우선순위

자동 배치 로직은 아래 우선순위로 처리한다.

1. 사용 불가 좌석, 결석 학생 제외
2. 고정 좌석 반영
3. 만나면 안 되는 학생 분리
4. 성별 배치 모드 반영
5. 나머지 자리 랜덤 배치

## 5.5 금지 / 선호 규칙

교사가 학생 간 관계 조건을 설정할 수 있어야 한다.

필수 규칙:

- 만나면 안 되는 학생
- 같은 모둠 금지
- 같은 짝 금지

확장 가능 규칙:

- 가까이 앉으면 안 됨
- 반드시 떨어뜨리기
- 앞자리 선호
- 교탁 근처 금지

규칙 위반이 발생하면:

- 위반 학생을 색상으로 표시
- 어떤 규칙이 충돌했는지 알려주기
- `그래도 배치 적용` 또는 `다시 배치` 선택 가능

## 5.6 저장 / 불러오기

- 학년 / 반 / 교실 단위로 저장
- 변경 사항 자동저장
- 최근 배치 불러오기
- 이전 배치 복사 후 새 배치 만들기
- 저장본 스냅샷 만들기
- 좌석 레이아웃 템플릿 저장
- 학생 명단 저장

저장 대상:

- 학생 명단
- 좌석 위치
- 모둠 구성
- 고정 좌석
- 금지 규칙
- 인쇄 옵션

저장 동작 원칙:

- 편집 중 변경 사항은 자동저장한다.
- 사용자는 현재 배치를 별도 이름으로 저장할 수 있어야 한다.
- 반별로 여러 저장본을 보관할 수 있어야 한다.
- 최근 작업 반을 빠르게 다시 열 수 있어야 한다.

## 5.7 인쇄 / 내보내기

- 교사용 인쇄
- 학생용 인쇄
- PDF 저장
- 이미지 저장

인쇄 옵션 예시:

- 이름 표시 / 번호 표시 / 둘 다 표시
- 결석 학생 숨기기
- 모둠 테두리 표시 여부
- 교탁 표시 여부

## 6. 화면 구조 제안

### 화면 1. 반 목록 화면

- 학년 선택
- 반 선택
- 저장된 배치 목록
- 새 배치 만들기

### 화면 2. 편집기 화면

- 상단: 반 정보, 저장, 인쇄, 관점 전환

- 중앙: 좌석 캔버스
- 우측: 프리셋, 자동 배치 옵션, 모둠 설정

### 화면 3. 인쇄 미리보기

- 교사용 / 학생용 전환
- 페이지 레이아웃 확인
- PDF 저장 / 인쇄

## 8. MVP 범위

첫 버전에서는 아래 기능만 우선 구현한다.

1. 반/학생 명단 저장
2. 좌석 프리셋 생성
3. 좌석 드래그 편집
4. 모둠 묶기 및 테두리 표시
5. 교사 관점 / 학생 관점 전환
6. 완전 랜덤 / 고정 포함 랜덤
7. 만나면 안 되는 학생 규칙
8. 성별 배치 모드 3종
9. 저장 / 불러오기
10. 교사용 / 학생용 인쇄

## 9. 후속 확장 기능

- CSV 업로드
- 이전 배치와의 유사도 제어
- 남녀 비율 자동 균형
- 학생 메모 기반 추천 배치
- 모바일 최적화
- 반별 배치 이력 비교
- 학부모 공개용 보기 모드

## 10. 구현 시 주의사항

- 드래그 편집은 직관적이어야 한다.
- 랜덤 배치는 `완전 무작위`보다 `교사가 통제 가능한 무작위`에 초점을 맞춰야 한다.
- 성별 조건은 보조 규칙으로 취급해야 하며, 다른 안전 규칙보다 우선되면 안 된다.
- 인쇄 레이아웃은 실제 교실에서 한눈에 보여야 하므로 글자 크기와 여백을 크게 잡아야 한다.
- 전담 교사 사용성이 중요하므로 `빠른 불러오기`와 `복제 저장`이 필수다.
